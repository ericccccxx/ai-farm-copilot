# 30-Day Launch Plan

## Goal

Launch AI Farm Copilot as a credible open-source project with enough substance to attract GitHub stars, early pilots, hardware conversations, and investor interest.

## Week 1: Build the Public Skeleton

- Finalize repository name and README.
- Publish the first agent skill.
- Add four scenario examples.
- Add local demo.
- Add safety disclaimer.
- Add data templates.
- Create first launch screenshots.

Outcome: the project is ready for private review.

## Week 2: Make It Look Real

- Record a short demo video.
- Add 3-5 realistic inspection reports.
- Add hardware integration diagrams.
- Test with phone photos.
- Test with drone or fixed camera photos if available.
- Improve README based on first reactions.

Outcome: the repository feels usable, not just conceptual.

## Week 3: Private Pilot

- Ask 5-10 people to test it.
- Prioritize vineyard, orchard, golf turf, hobby farm, and garden users.
- Track where they get confused.
- Collect photos, field notes, and report feedback.
- Ask whether they would use a hosted dashboard or hardware kit.

Outcome: first evidence of real-world demand.

## Week 4: Public Launch

- Publish the repository.
- Post launch story on LinkedIn.
- Share with open-source, agriculture, robotics, drone, and AI agent communities.
- Ask early users to star, fork, and submit issues.
- Start a public pilot waitlist.

Outcome: first public distribution loop.

## Metrics to Track

- GitHub stars.
- Forks.
- Skill installs.
- Demo visits.
- Pilot requests.
- Photos analyzed.
- Field reports generated.
- Managed areas logged.
- Hardware integration leads.
